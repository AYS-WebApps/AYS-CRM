<objective>
## Goal
Add a two-tier notes system to the client detail page: general client notes (not tied to any project) and per-project notes (nested inside each project card). Notes are append-only (add + delete). All notes for a client are fetched in a single query, then filtered by context in the UI.

## Purpose
Completes Phase 4. The AYS team can now leave running notes on any client (general reminders, contact history, preferences) and project-specific notes (measurements, delivery instructions, event details) â€” keeping all context in one place without cluttering the pipeline fields.

## Output
- `note-actions.ts` â€” `addNote`, `deleteNote`, `getNotesByClient` server actions
- `NotesSection.tsx` â€” reusable Client Component (general + project contexts)
- Updated `page.tsx` â€” fetches all client notes in parallel with projects
- Updated `ClientDetail.tsx` â€” adds `notes` prop + renders general NotesSection below projects
- Updated `ProjectsSection.tsx` â€” adds `notes` prop + renders per-project NotesSection inside each project card (view mode)
</objective>

<acceptance_criteria>

## AC-1: Add General Client Note
```gherkin
Given I am on a client detail page in view mode
When I type text in the general "Notes" section textarea and click "Add Note"
Then the note appears at the top of the notes list with its timestamp
And the textarea clears after submission
And the submit button is disabled while the request is pending
```

## AC-2: Add Project-Specific Note
```gherkin
Given a client has at least one project
When I type a note in that project's "Notes" subsection and click "Add Note"
Then the note appears under that project with its timestamp
And the note does NOT appear in the general notes section
```

## AC-3: Delete Note (With Confirmation)
```gherkin
Given a note exists in any section
When I click the "Delete" button on a note
Then an inline confirmation replaces the note row: "[Confirm delete?] [Yes] [Cancel]"
When I click "Yes"
Then the note is permanently removed from the list
```

## AC-4: Notes Persist Across Navigation
```gherkin
Given I have added notes to a client
When I navigate away and return to the client detail page
Then all notes are still visible in the correct section (general or per-project)
```

## AC-5: Empty States
```gherkin
Given a client or project has no notes
When I view the Notes section
Then "No notes yet." is shown above the input area
```

## AC-6: Notes Ordered Newest First
```gherkin
Given a client has multiple notes in a section
When I view those notes
Then they appear in reverse chronological order (newest at top)
```

    ```ts
    function handleAdd(e: React.FormEvent) {
      e.preventDefault()
      setAddError(null)
      if (!content.trim()) {
        setAddError('Note cannot be empty.')
        return
      }
      startTransition(async () => {
        const result = await addNote(clientId, content, projectId ?? null)
        if ('error' in result) {
          setAddError(result.error)
        } else {
          setContent('')
          router.refresh()
        }
      })
    }
    ```

    **handleDelete(noteId: string)** â€” called on confirm:
    ```ts
    startTransition(async () => {
      const result = await deleteNote(noteId, clientId)

      } else {
        setDeletingId(null)
        router.refresh()
      }
    })
    ```
