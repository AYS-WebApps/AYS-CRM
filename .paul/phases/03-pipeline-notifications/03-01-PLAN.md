<objective>
## Goal
Add a Projects section to the client detail page. Each project (event/booking) has a title, event date, pipeline stage, next action, and next action due date. Projects are created, edited, and deleted inline — no separate route.

## Purpose
Introduces the pipeline concept into the CRM. The AYS team can now attach one or more bookings to any client, set each booking's stage (New Lead → Confirmed, etc.), and record what needs to happen next and by when. This is the data foundation Phase 3-02 will surface on the dashboard as alerts.

## Output
- `ProjectWithStage` type (project row joined with stage name + color)
- Server actions: `getProjectsByClient`, `getStages`, `createProject`, `updateProject`, `deleteProject`
- `ProjectsSection` Client Component — list of project cards with inline create/edit/delete
- `CreateProjectModal` — modal form for title + event date; stage auto-set to New Lead on create
- Client detail page updated to load and pass projects + stages
</objective>

    <!-- audit-added: G3 — normalize empty strings to null for all optional fields -->
    - Extract and normalize:
      ```ts
      const title = (formData.get('title') as string).trim()
      const event_date = (formData.get('event_date') as string).trim() || null
      const pipeline_stage_id = (formData.get('pipeline_stage_id') as string).trim() || null

      const next_action_due_at = (formData.get('next_action_due_at') as string).trim() || null
      ```

    - If `data` is empty array: return `{ error: 'Project not found. It may have been deleted.' }`
    - On DB error: return `{ error: 'Failed to update project. Please try again.' }`
    - On success: `revalidatePath(\`/dashboard/clients/${clientId}\`)`, `revalidatePath('/dashboard')`, return `{ success: true }`

    ```ts
    function formatDate(dateString: string | null): string {
      if (!dateString) return '—'
      // For DATE columns (YYYY-MM-DD): parse as local time to avoid UTC offset shift
    ```
