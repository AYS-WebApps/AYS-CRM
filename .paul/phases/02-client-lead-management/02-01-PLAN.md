<task type="auto">
  <name>Task 1: Create server actions for client list and create</name>
  <files>src/app/(dashboard)/dashboard/clients/actions.ts</files>
  <action>
    Create server actions file at `src/app/(dashboard)/dashboard/clients/actions.ts`:

    **`getClients()`** — fetch all clients, ordered by created_at DESC:
    ```ts
    'use server'
    import { createClient } from '@/lib/supabase/server'
    import type { Client } from '@/lib/types'

    export async function getClients(): Promise<Client[]> {
      const supabase = await createClient()
      const { data, error } = await supabase
        .from('clients')
        .select('*')
        .order('created_at', { ascending: false })
      if (error) throw new Error(error.message)
      return data
    }
    ```

    **`addClient(formData: FormData)`** — validate and insert:
    1. Extract: name (trim), phone (trim, nullable), email (trim, lowercase, nullable), source
    2. Validate: name must be non-empty after trim
    3. Validate: if email provided, must match basic pattern `^[^@\s]+@[^@\s]+\.[^@\s]+$`
    4. Insert via Supabase — catch PostgreSQL unique violation (error code `23505`) and return
       `{ error: 'A client with this email already exists.' }`
    5. On other DB errors: log server-side with timestamp (no PII), return generic message
    6. On success: call `revalidatePath('/dashboard/clients')` AND `revalidatePath('/dashboard')`
       (revalidates dashboard stat cards too), then return `{ success: true }`

    Return type: `{ success: true } | { error: string }`

    Source values passed from form must be one of: 'website' | 'direct' | 'referral' | 'whatsapp' | 'other'
    Default to 'direct' if not provided or invalid.

    Avoid: Throwing unhandled errors to the client — always return typed result object.
    Avoid: Logging email addresses in error output.
  </action>
