<objective>
## Goal
Create all core database tables in Supabase with RLS policies, seed pipeline stages, and generate TypeScript types — giving every future phase a typed, secured data layer to build on.

## Purpose
Without the schema, no feature (clients, pipeline, notes, Gmail capture) can be built. This plan establishes the single source of truth for the data model. Getting relationships right here (especially client → projects → notes) prevents schema rewrites in later phases.

## Output
- SQL migration file with 4 tables: `pipeline_stages`, `clients`, `projects`, `notes`
- RLS policies enabling all authenticated users full access (team CRM — no per-user isolation in v0.1)
- `updated_at` auto-update triggers on mutable tables
- 6 seeded pipeline stages (New Lead → Contacted → Quoted → Confirmed → Completed → Cancelled)
- TypeScript type definitions matching the schema
</objective>

<task type="auto">
  <name>Task 1: Write SQL migration and TypeScript type definitions</name>
  <files>supabase/migrations/001_initial_schema.sql, src/lib/types/database.ts, src/lib/types/index.ts</files>
  <action>
    **Create `supabase/migrations/001_initial_schema.sql`:**

    Table: `pipeline_stages` (lookup, seeded — never user-created)
    - id UUID PK default gen_random_uuid()
    - name TEXT NOT NULL
    - sort_order INTEGER NOT NULL
    - color TEXT NOT NULL DEFAULT '#6B7280'
    - created_at TIMESTAMPTZ NOT NULL DEFAULT now()

    Table: `clients`
    - id UUID PK default gen_random_uuid()
    - name TEXT NOT NULL CHECK (char_length(trim(name)) > 0)   <!-- audit-added: no empty names -->
    - phone TEXT (nullable — some inquiries have no phone)
    - email TEXT (nullable — some inquiries have no email)
      <!-- audit-added: email format check when present -->
      CHECK (email IS NULL OR email ~* '^[^@\s]+@[^@\s]+\.[^@\s]+$')
    - source TEXT NOT NULL DEFAULT 'direct'
      — allowed values: 'website', 'direct', 'referral', 'whatsapp', 'other'
      — use a CHECK constraint: CHECK (source IN ('website','direct','referral','whatsapp','other'))
    - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
    - updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

    Table: `projects` (one client → many events/bookings)
    - id UUID PK default gen_random_uuid()
    - client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE
    - title TEXT NOT NULL (e.g., "Smith Wedding June 2026")
    - event_date DATE (nullable — may not be known yet)
    - pipeline_stage_id UUID REFERENCES pipeline_stages(id) ON DELETE SET NULL
    - next_action TEXT (nullable — what needs to happen next)
    - next_action_due_at TIMESTAMPTZ (nullable — when it's due)
    - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
    - updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

    Table: `notes`
    - id UUID PK default gen_random_uuid()
    - client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE
    - project_id UUID REFERENCES projects(id) ON DELETE CASCADE (nullable — NULL = general client note)
    - content TEXT NOT NULL CHECK (char_length(trim(content)) > 0)  <!-- audit-added: no empty notes -->
    - created_by UUID REFERENCES auth.users(id) ON DELETE SET NULL
    - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
    - updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

    **Seed pipeline_stages:**
    ```sql
    INSERT INTO pipeline_stages (name, sort_order, color) VALUES
      ('New Lead',   1, '#3B82F6'),
      ('Contacted',  2, '#8B5CF6'),
      ('Quoted',     3, '#F59E0B'),
      ('Confirmed',  4, '#10B981'),
      ('Completed',  5, '#6B7280'),
      ('Cancelled',  6, '#EF4444');
    ```

    **updated_at trigger** (apply to clients, projects, notes):
    ```sql
    CREATE OR REPLACE FUNCTION set_updated_at()
    RETURNS TRIGGER AS $$
    BEGIN
      NEW.updated_at = now();
      RETURN NEW;
    END;
    $$ LANGUAGE plpgsql;
    ```
    Create trigger on each table: BEFORE UPDATE FOR EACH ROW EXECUTE FUNCTION set_updated_at()

    <!-- audit-added: FK indexes — PostgreSQL does NOT auto-index FK columns -->
    **Indexes — add after table creation:**
    ```sql
    -- Prevent duplicate clients on same email (partial — email is nullable)
    CREATE UNIQUE INDEX clients_email_unique ON clients(email) WHERE email IS NOT NULL;

    -- FK performance indexes (full table scan without these)
    CREATE INDEX idx_projects_client_id ON projects(client_id);
    CREATE INDEX idx_projects_pipeline_stage_id ON projects(pipeline_stage_id);
    CREATE INDEX idx_projects_next_action_due_at ON projects(next_action_due_at) WHERE next_action_due_at IS NOT NULL;
    CREATE INDEX idx_notes_client_id ON notes(client_id);
    CREATE INDEX idx_notes_project_id ON notes(project_id) WHERE project_id IS NOT NULL;
    ```

    <!-- audit-added: pipeline stage default responsibility -->
    **Application-layer rule for pipeline stage:**
    `pipeline_stage_id` cannot be defaulted in DDL (FK to a seeded row whose UUID is unknown at migration time).
    When inserting a new project, the application MUST query `pipeline_stages` for the 'New Lead' row and set its id.
    Projects with NULL pipeline_stage_id are invisible in the pipeline board (Phase 3). Document this invariant.

    **RLS policies** — enable RLS on all 4 tables, then:
    - pipeline_stages: SELECT only for authenticated (it's a lookup table, no writes from app)
    - clients, projects, notes: ALL operations for authenticated users
    Pattern: `USING (true) WITH CHECK (true)` — every authenticated user can access all records (team CRM)

    **Create `src/lib/types/database.ts`:**
    Define a `Database` type following Supabase's generated types shape:
    ```ts
    export type Database = {
      public: {
        Tables: {
          pipeline_stages: { Row: PipelineStageRow; Insert: PipelineStageInsert; Update: PipelineStageUpdate }
          clients: { Row: ClientRow; Insert: ClientInsert; Update: ClientUpdate }
          projects: { Row: ProjectRow; Insert: ProjectInsert; Update: ProjectUpdate }
          notes: { Row: NoteRow; Insert: NoteInsert; Update: NoteUpdate }
        }
      }
    }
    ```
    Define each Row/Insert/Update type:
    - Row = all columns present, nullable columns use `string | null`
    - Insert = required columns required, optional columns optional (?)
    - Update = all columns optional (Partial)

    Use `ClientSource` as a union type: `'website' | 'direct' | 'referral' | 'whatsapp' | 'other'`

    **Create `src/lib/types/index.ts`:**
    Re-export convenience aliases:
    ```ts
    export type { Database } from './database'
    // Row type aliases for easy use in components
    export type PipelineStage = Database['public']['Tables']['pipeline_stages']['Row']
    export type Client = Database['public']['Tables']['clients']['Row']
    export type Project = Database['public']['Tables']['projects']['Row']
    export type Note = Database['public']['Tables']['notes']['Row']
    export type ClientSource = 'website' | 'direct' | 'referral' | 'whatsapp' | 'other'
    ```

    Avoid: Adding `venue`, `equipment`, or any inventory fields — out of scope for v0.1
    Avoid: Soft deletes on any table — use CASCADE hard deletes for now (simpler, can add later)
    Avoid: User-specific RLS — all team members share access in v0.1
  </action>
